
# 🧠 CONTEXT — AI-Powered Scientific Research Assistant

## Project Overview
This project is a **Streamlit-based web application** that helps researchers verify scientific claims against peer-reviewed literature.
The user enters a claim (e.g. "Coffee reduces Alzheimer's risk"), the app fetches relevant papers from Semantic Scholar,
analyzes each one with an LLM (OpenAI GPT-4o), and returns a structured verdict: SUPPORTS / CONTRADICTS / NEUTRAL.

---

## 🗂️ Current Stack
- **Frontend**: Streamlit (Python)
- **Backend logic**: Python modules
- **LLM**: Multi model via Groq API (3 keys rotation with 7 models)
- **Paper search**: Semantic Scholar API
- **Full text retrieval**: Unpaywall API (PDF) + PyMuPDF (text extraction)
- **Caching**: Local JSON-based cache (`cache.py`) — per paperId + per claim
- **Deployment**: Local / Streamlit Cloud

## 🚧 What Needs To Be Done

### Step 2 — Background Full Text Enrichment (Streamlit)
**Goal**: After results are displayed, silently fetch PDFs in background threads.
**Details**:
- Launch a `threading.Thread(daemon=True)` after `st.write(results)`
- Thread calls `fulltext.fetch(pid, doi)` for each paper not yet in cache
- Stores result via `cache.set_fulltext(pid, text)`
- Sidebar shows `🔄 Enriching X papers in background...` badge
- On next search for same claim → full texts are used automatically

**Files to modify**: `app.py`, optionally `cache.py`

### Step 5 — Next.js Frontend (Full Rewrite / Separate Frontend)
**Goal**: Replace Streamlit with a proper web UI in Next.js for a production-grade experience.
**Details**:

#### Architecture
- **Frontend**: Next.js 14+ (App Router), TailwindCSS, shadcn/ui
- **Backend**: FastAPI (Python) — expose current logic as REST API endpoints
- **Communication**: REST or tRPC

#### API Endpoints to build (FastAPI)
| Method | Route | Description |
|--------|-------|-------------|
| POST | `/search` | Search papers by claim |
| GET | `/paper/{id}` | Get paper details + full text |
| POST | `/analyze` | Analyze a paper against a claim |
| POST | `/chat` | Chat about a specific paper |
| GET | `/history` | Get user search history |
| POST | `/bookmark` | Save a paper |

#### Next.js Pages to build
| Route | Description |
|-------|-------------|
| `/` | Home — search bar, hero |
| `/results` | Results page — paper cards + verdicts |
| `/paper/[id]` | Individual paper view + chat |
| `/dashboard` | User history + saved papers |

#### UI Components (shadcn/ui recommended)
- `ClaimSearchBar` — main input with autocomplete
- `PaperCard` — title, authors, verdict badge, confidence bar
- `VerdictBadge` — SUPPORTS / CONTRADICTS / NEUTRAL with colors
- `PaperChatDrawer` — side panel with chat interface
- `HistoryTimeline` — dashboard list of past searches
