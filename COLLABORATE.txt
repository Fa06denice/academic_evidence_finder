# 🧠 COLLABORATE — AI-Powered Scientific Research Assistant

## Project Overview
**Academic Evidence Finder** is a Streamlit web application that helps researchers
verify scientific claims and explore academic literature using AI.

Three core modes:
- **Claim Verifier** — enter a claim → fetch papers → LLM verdict: SUPPORTED / CONTRADICTED / MIXED
- **Literature Review** — enter a topic → generate a structured academic review from retrieved papers
- **Paper Summarizer** — search a topic → get structured summaries of individual papers

---

## 🗂️ Current Stack

| Layer | Technology |
|---|---|
| Frontend | Streamlit (Python) |
| LLM | Groq API — multi-model rotation (primary: `moonshotai/kimi-k2-instruct`) |

| Full text | Unpaywall API + PyMuPDF — available in `scholar_client.py`, not in main pipeline yet |
| Caching | Local JSON (`paper_cache.json`) via `cache_manager.py` |

## ✅ What Is Done

### Core Pipeline
- [x] Claim → multi-query transform (3-5 Semantic Scholar queries via LLM)
- [x] Papers fetched, deduplicated, sorted by citation count
- [x] Relevance filter: `relevance_score >= 5` (papers below threshold excluded from verdict)
- [x] Each paper analyzed by LLM → verdict + confidence + evidence + key_finding
- [x] Overall verdict synthesized by LLM from all paper analyses
- [x] Literature Review mode: structured 4-section academic review
- [x] Paper Summarizer mode: 5-section structured summary per paper

### LLM / Analyzer
- [x] Multi-model rotation across 6+ Groq models — automatic failover on 429
- [x] Primary model: `moonshotai/kimi-k2-instruct`
- [x] Fallback chain: `qwen/qwen3-32b` → `llama-3.3-70b-versatile` → `llama-4-scout` → `gpt-oss-120b` → `gpt-oss-20b`
- [x] All prompts centralized in `analyzer.py` — V1 FINAL versions
- [x] Population mismatch cap: cross-population evidence capped at PARTIALLY_SUPPORTS
- [x] In vitro / animal model rule: mechanistic studies ≠ SUPPORTS for human claims
- [x] Absence/no-effect rule: "no effect", "no association" → CONTRADICTS when population matches
- [x] Query scoping: `_TOPIC_QUERIES` tightly scoped to prevent field drift
- [x] Claim validation: vague/empty claims rejected before pipeline

### Caching
- [x] Search results cached per query string (MD5 key)
- [x] LLM analyses cached per `paperId + claim_hash`
- [x] Summaries cached per `paperId`
- [x] Full text cached per `paperId` (for future use)
- [x] `paper_cache.json` excluded from git via `.gitignore`

### UI (Streamlit — current state)
- [x] 3-tab layout: Claim Verifier / Literature Review / Paper Summarizer
- [x] Color-coded verdict badges (green / yellow / red / grey)
- [x] Paper cards: title, authors, year, citations, evidence, explanation, key finding
- [x] Overall verdict banner with supporting / contradicting / neutral counts
- [x] Sidebar: model in use, cache stats, search parameters
- [x] Max papers slider, year filter

## 🏗️ Target Architecture (Production)

### Step 2 — FastAPI Backend
**Goal**: expose all Python logic as REST API endpoints.

| Method | Route | Description |
|---|---|---|
| POST | `/verify` | Full claim verification pipeline |
| POST | `/review` | Literature review generation |
| POST | `/summarize` | Paper summarization |
| GET | `/paper/{id}` | Paper details |

- Migrate `scholar_client.py`, `analyzer.py` into FastAPI routers
- Async with `httpx` + `asyncio`
- All prompts stay in `analyzer.py`

### Step 3 — Next.js Frontend
**Goal**: replace Streamlit with a production-grade web UI.

| Technology | Role |
|---|---|
| Next.js 14+ (App Router) | Framework |
| TailwindCSS | Styling |
| shadcn/ui | UI components (cards, badges, dialogs) |
| Zustand | State management |
| TanStack Query | Fetching + client-side cache |
| Framer Motion | Animations |
| NextAuth.js | Auth (Google OAuth / magic link) |

**Pages**:
- `/` — home, search bar, hero
- `/results` — paper cards + verdict
- `/paper/[id]` — individual paper view
- `/dashboard` — search history + saved papers

### Step 4 — Database & Cache (Production)

| Current | Production |
|---|---|
| JSON file cache | **Redis** — fast analysis cache |
| No DB | **MongoDB Atlas** + Beanie (ODM async) |

## 🤝 Collaboration Rules

- **All LLM prompts in `analyzer.py` only** — never duplicate or inline prompts elsewhere
- **Model rotation in `analyzer.py` only** — never hardcode model names in `app.py`
- **Cache keys**: `{paperId}_{claim_hash}` pattern — do not change
- **UI language**: French for all user-facing labels and messages
- **Code language**: English (variables, functions, comments, docstrings)
- **Relevance threshold**: `>= 5` — do not lower without testing
- **`paper_cache.json`** is gitignored — never commit it
